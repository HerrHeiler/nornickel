# Скажи мне, кто твой шлиф

Решение кейса по автоматической классификации руд на панорамных OM-снимках полированных шлифов. Система принимает изображение (TIFF/PNG/JPEG/BMP, вплоть до 10 000×10 000 пикселей), сегментирует сульфидные срастания и тальк, считает площадные доли и выдаёт вердикт по экспертным правилам — с цветной маской, таблицей метрик и текстовым заключением.

```
тальк > 10 % площади шлифа           → оталькованная руда
тальк ≤ 10 %, преобладают обычные    → рядовая руда
тальк ≤ 10 %, преобладают тонкие     → труднообогатимая руда
```

## Как посмотреть

```bash
pip install -r requirements.txt
streamlit run app.py
```

Загрузите любой снимок шлифа — через полминуты получите маску (зелёный — обычные срастания, красный — тонкие, синий — тальк), таблицу долей, заключение и карту уверенности. Работает и без GPU. Пакетный режим для партий: `python scripts/run_batch.py --input <папка> --output outputs`.

Развёрнутая версия: репозиторий деплоится на Streamlit Community Cloud как есть (веса модели, 25 МБ, лежат в `models/best.pt`). На бесплатном тарифе для панорам стоит выставить лимит 25–50 Мпикс в сайдбаре.

## Что внутри

Два взаимодополняющих бэкенда в одном пайплайне:

**U-Net + EfficientNet-B0** (`segmentation_models_pytorch`). Панорамы не влезают в память целиком, поэтому инференс тайловый: окна 1024² с перекрытием 256 пикселей, вероятности блендятся гауссовым окном — швов на стыках нет. Изображения сверх 100 Мпикс автоматически даунскейлятся, так что 5-минутный лимит из ТЗ выполняется на любом железе. Побочный продукт блендинга — попиксельная карта уверенности, которая подсвечивает геологу спорные участки.

**CV-бейзлайн** работает без GPU и без весов: сульфиды — яркая отражающая фаза (порог адаптивный, от медианы матрицы, поэтому выдерживает и тёмные панорамы), тальк — плотность мелких тёмных вкраплений в скользящем окне, срастания разделяются по размеру зерна и коэффициенту замещения. Он же дал weak labels для обучения нейросети.

## Как мы получили разметку

Готовых масок в данных нет. Единственная пиксельная разметка — 42 фото оталькованных руд, где области талька обведены синей линией. Причём линии оказались не замкнутыми контурами, а разделителями: петли, отсечённые углы кадра, коридоры между двумя линиями. Мы восстановили зоны автоматически: выделение линии по цвету, дорисовка открытых концов до границы кадра (через скелетизацию), разбиение кадра на области — наибольшая считается вмещающей породой, остальные тальком. Все 42 пары сконвертировались без ручных правок, каждую проверили глазами.

Для сульфидов пиксельной разметки нет вовсе, поэтому обучали на weak labels от CV-бейзлайна и добавили 136 фото рядовых и труднообогатимых руд как негативы по тальку. Без негативов модель, видевшая только оталькованные шлифы, находила тальк на любой матрице — это дало худшие цифры из всех наших экспериментов, см. таблицу ниже.

Оценивались честно: ~1180 фото в датасете разложены по папкам сортов, это метки уровня изображения. Прогоняем систему по ним и сравниваем вердикт с папкой (`scripts/evaluate_grades.py`) — это и есть бизнес-метрика кейса, а не IoU сегментации.

## Результаты

Важная находка при доводке: первые версии оценивались на выборке, частично пересекавшейся с обучением (фото ч1 участвовали в обучении как негативы), и цифры были завышены. Ниже — честный протокол: обучение не видит фото из оценочной выборки вообще; финальный замер — 120 фото ч2 (по 40 на сорт), полностью отделённых от обучения и от калибровки.

Что мы перебрали (короткие прогоны, единый чистый срез):

| Эксперимент | Macro-F1 |
|---|---|
| Исходная модель (обучение: 42 оталькованных + негативы ч1) | 0.315 |
| + сильные фотометрические искажения (экспозиция, гамма, шум, размытие, тени) | 0.352 |
| + ~200 фото ч2 с folder-prior разметкой срастаний | 0.438 — срастания резко лучше, тальк «задавлен» разбавлением |
| + ребаланс талька (вес класса ×4, оверсэмплинг ×3) | лучший вариант |

Ещё два урока. Первый: отбор чекпоинта по общему mIoU против weak labels маскирует коллапс талька (лёгкий фон доминирует в среднем) — отбираем по среднему IoU трёх рудных классов. Второй: U-Net полностью «слепнет» на тальке у снимков с камеры, которой не было в обучении (половина оталькованных ч2 — предсказание талька ровно 0 %), и никакая калибровка нуля не исправит.

**Гибридное измерение талька** — ответ на второй урок: классический детектор рассеянной тёмной фазы (плотность вкраплений) не зависит от камеры и уверенно срабатывает там, где нейросеть молчит. Финальная доля талька = максимум двух независимо калиброванных измерений (нейросетевого и классического); маска и остальные классы остаются за U-Net.

Итог на чистых фото ч2, не участвовавших ни в обучении, ни в подборе порогов друг друга (150 фото, единые для сравнения):

| Модель | Accuracy | Macro-F1 | Бинарный F1 по типу срастаний |
|---|---|---|---|
| Было (первая версия) | 0.475 | 0.393 | — |
| U-Net v2 (данные ч2 + ребаланс) | 0.658 | 0.645 | 0.75 |
| **Гибрид: U-Net + CV-тальк** | **0.680** | **0.678** | **0.833** |

Оценка переносимости калибровки (2-fold между независимыми срезами): macro-F1 ≈ 0.58 — честный нижний ориентир; верхний — 0.68–0.74 на срезе калибровки.

Калибровка (`scripts/calibrate.py` → `models/calibration.json`) — принципиальный момент: экспертные правила (порог 10 % талька, «преобладание» срастаний) мы не трогаем, калибруются только измерения. Плата за гибрид: ошибка *площади* талька на ч1-парах выросла до MAE 16.8 пп (классическая ветвь переоценивает на снимках стиля ч1) — гибрид оптимизирует правило 10 %, а не площадь; для площади используется нейросетевое измерение (MAE 11.6 пп).

## Соответствие требованиям ТЗ

- **Производительность.** Панорама 13 712×10 798: 94 с на CPU бейзлайном, около 4 минут нейросетью на бюджетном GPU (GTX 1650); авто-даунскейл гарантирует лимит 5 минут.
- **Надёжность.** Проверяли синтетическими «сложными случаями» (`scripts/robustness_check.py`): царапины, загрязнение шлифа, размытие, недодержка ×0.5 — вердикт стабилен (устойчивость к недодержке дало обучение с сильными фотометрическими искажениями). Слабое место гибрида — сильный градиент освещения (классическая ветвь использует глобальные пороги): дрейф талька до 10 пп; смягчение — съёмка с равномерной подсветкой либо отключение CV-ветви (`cv_talc_gain: 0`).
- **Точность.** F1 по типу срастаний (бинарный, рядовые/тонкие) — 0.833 при цели ≥ 0.90; macro-F1 по трём сортам 0.68. Доля талька: MAE 11.6 пп (нейросетевое измерение) при цели ±3 пп — эксперт обводит зоны целиком, модель меряет рассеянную фазу. Оба зазора упираются в отсутствие пиксельной разметки; решение встроено — см. active learning ниже.
- **Интерфейс.** Streamlit без ML-жаргона: загрузил — получил вердикт. Зум, карта уверенности, экспорт CSV/JPEG. Ручная коррекция маски: скачал индексную маску, поправил кистью в любом редакторе, загрузил обратно — метрики пересчитываются, пара (снимок, маска) сохраняется в `data/corrections` для дообучения.
- **Безопасность.** Полностью локально и офлайн: веса грузятся из чекпоинта, никаких обращений в сеть.

## Структура

```
app.py                          Streamlit-интерфейс
src/
  pipeline.py                   оркестрация: загрузка → сегментация → метрики → экспорт
  model.py                      U-Net + CV-бейзлайн
  tiling.py                     тайловый инференс с блендингом
  preprocessing.py              загрузка форматов, CLAHE, маска области шлифа
  metrics.py                    площадные доли + экспертные правила
  train.py                      обучение (Dice+CE, аугментации)
scripts/
  prepare_dataset.py            разметка → обучающие кропы (+ --debug для QA-оверлеев)
  evaluate_grades.py            оценка сортности на размеченных папках
  calibrate.py                  калибровка измерений по экспертным данным
  robustness_check.py           тест на «сложных случаях»
  run_batch.py                  пакетная обработка
models/best.pt                  веса (без них работает CV-бейзлайн)
```

## Что дальше

Главный резерв точности — заменить weak labels настоящей разметкой, и цикл для этого уже в продукте: геолог правит маски прямо из интерфейса, исправленные пары накапливаются для дообучения. По нашей оценке, 20–30 размеченных фото закрывают и ±3 пп по тальку, и F1 0.90 по срастаниям.

Второе направление — претрейн сульфидной сегментации на открытом датасете [LumenStone S1](https://imaging.cs.msu.ru/en/research/geology/lumenstone) (МГУ): 84 снимка полированных шлифов в отражённом свете с попиксельными масками семи сульфидных минералов — ближайший по домену открытый источник. Дальше — transfer learning на другие типы руд и оборудование, экспорт в ГИС-форматы.
